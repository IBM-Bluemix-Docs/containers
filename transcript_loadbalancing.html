Transcript for container_group_loadbalancing.gif

A person is displayed alongside a grey box labeled A. The person represents a user and box A represents a container group.

Inside box A, a blue box is displayed and labeled B, which represents the load balancer for the container group.

Also inside box A, three boxes are displayed, which represents three instances within the container group.

A request is sent from the user through the load balancer, and then to the first container group instance. This request spins and when it completes, it fades.

A communication is sent from the user through the load balancer, and then to the second instance. This request spins and does not complete. The request continues to spin.

A communication is sent from the user through the load balancer, and then to the third instance. This request spins and when it completes, it fades.

A request is sent from the user through the load balancer, and then to the first instance. This request spins and when it completes, it fades.

A communication is sent from the user through the load balancer, and then to the second instance. This request spins and waits for the stuck request to complete. When the stuck request completes and fades, the next request spins and when it completes, it fades.

A communication is sent from the user through the load balancer, and then to the third instance. This request spins and when it completes, it fades.
